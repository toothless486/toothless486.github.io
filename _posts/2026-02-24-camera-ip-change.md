> 카메라에 접근이 안 된다는 문제를 해결하다가, IP 설정의 기초부터 다시 짚게 됐다.

---

## 발단

카메라에 접근이 안 된다고 하니, 동료가 원인과 해결 방향을 설명해줬다.

내부망에 연결된 상태에서는 네트워크 대역이 달라 카메라 조회가 막힐 수 있다고 했다.
해결 방법으로는 PoE 스위치의 상단 연결을 제거한 뒤, 카메라의 기본 IP와 같은 대역으로 PC IP를 맞추는 것을 제안했다.
예를 들어 하이크비전 카메라의 기본 IP가 `172.16.10.50`라면, PC는 `172.16.10.51`로 설정하는 식이다.

카메라에 접근이 안 되는 이유가 **"IP 대역이 달라서"** 라는 건데, 처음 들으면 감이 잘 안 온다.
이번 기회에 IP, Gateway, Subnet Mask 개념을 제대로 정리해봤다.

---

## IP 주소란?

IP 주소는 네트워크에서 **각 장치를 구분하는 주소**다.
사람으로 치면 집 주소와 같다. `172.16.10.50` 같은 숫자 4개로 이루어진 형태가 일반적이다.

카메라도 네트워크 장비이므로 고유한 IP 주소를 가진다.
PC가 카메라에 접근하려면 이 IP를 알아야 한다.

---

## Subnet Mask — "같은 동네"인지 판별하는 기준

`255.255.255.0` 이라는 서브넷 마스크는 **IP 주소에서 어디까지가 "네트워크 주소"이고, 어디부터가 "개별 장치 주소"인지** 구분해준다.

예를 들어:

| 장치 | IP 주소 | 서브넷 마스크 |
|------|---------|--------------|
| 카메라 | 172.16.10.50 | 255.255.255.0 |
| PC | 172.16.10.51 | 255.255.255.0 |

서브넷 마스크 `255.255.255.0`은 앞 세 자리(`192.168.1`)가 **네트워크 주소**, 마지막 자리가 **장치 번호**임을 의미한다.
카메라와 PC 모두 `172.16.10.x` 대역이므로 **같은 네트워크, 즉 "같은 동네"** 에 있다.

반대로 카메라가 `172.16.10.50`인데 PC가 `10.10.0.100`이라면 서로 다른 네트워크여서 직접 통신이 불가능하다.

---

## Gateway — "다른 동네"로 나가는 출구

Gateway(게이트웨이)는 **내 네트워크 밖으로 나갈 때 거치는 관문**이다.
보통 공유기나 라우터가 이 역할을 한다.

```
[카메라] → [게이트웨이(공유기)] → [인터넷 or 다른 네트워크]
```

같은 대역 안의 장치끼리는 게이트웨이 없이 직접 통신하지만,
외부 네트워크로 나가려면 반드시 게이트웨이를 거쳐야 한다.

예를 들어:

- 카메라 IP: `172.16.10.50`
- 게이트웨이: `172.16.10.1` (공유기)
- 외부(인터넷)로 나가는 요청 → 공유기가 중계

---

## 카메라 IP를 바꾸는 이유

### 상황 1: PC와 대역이 달라서 접근 자체가 안 될 때

카메라가 설치된 환경(예: 사무실 내부망)과 접속하려는 PC의 네트워크 대역이 다르면,
**같은 네트워크가 아니기 때문에 IP로 직접 접근이 불가능하다.**

이때 해결책은 두 가지다:

1. PC의 IP를 카메라와 같은 대역으로 맞춘다.
2. 카메라의 IP를 PC와 같은 대역으로 바꾼다.

동료가 말한 방법은 1번이다.
PoE 스위치의 상단 연결(내부망 연결)을 제거하고, PC와 카메라를 직접 같은 대역으로 연결하면
내부망의 간섭 없이 카메라에 바로 접근할 수 있다.

### 상황 2: 외부에서 카메라에 접근하고 싶을 때

카메라가 내부망(`192.168.x.x` 같은 사설 IP)에 있으면 인터넷에서 직접 접근할 수 없다.
사설 IP는 내부에서만 유효한 주소이기 때문이다.

외부 접속을 가능하게 하려면 크게 두 가지 방법이 있다:

#### 방법 1: 포트 포워딩

```
[외부 사용자] → [공인 IP:포트] → [공유기] → [카메라 사설 IP:포트]
```

공유기에서 특정 포트로 들어오는 요청을 카메라의 IP로 전달(포워딩)하는 방법.
카메라 IP 자체를 바꾸는 게 아니라, **공유기가 중간에서 연결을 연결해주는** 구조다.

#### 방법 2: 카메라를 공인 IP 대역으로 이동

카메라를 ISP에서 할당받은 공인 IP 대역에 직접 배치하는 방법.
보안상 위험할 수 있어 실무에서는 방법 1을 많이 쓴다.

## 카메라 IP를 모를 때 — 랜선 + 탐색 프로그램으로 찾는 법

### 상황

카메라 IP를 바꾸려면 먼저 **현재 IP가 뭔지** 알아야 한다.
그런데 카메라에는 화면이 없고, 설정 파일에 기록이 없는 경우도 많다.

이때 노트북에 랜선을 꽂아 카메라와 직접 연결하고, 제조사 탐색 프로그램(플러그인)을 실행하면
프로그램이 카메라의 IP를 자동으로 찾아서 화면에 보여준다.

왜 이게 가능한 걸까?

### 원리: UDP 브로드캐스트 탐색

네트워크에는 **브로드캐스트(broadcast)** 라는 특수한 전송 방식이 있다.
일반 통신이 "특정 주소의 장치에게 보내는 편지"라면,
브로드캐스트는 **"같은 네트워크에 있는 모든 장치에게 동시에 보내는 방송"** 이다.

탐색 프로그램은 이 방식을 이용한다:

```
[탐색 프로그램] → "여기 누구 있어?" (브로드캐스트, 255.255.255.255)
                        ↓ 같은 네트워크의 모든 장치에게 전달
[카메라] → "저 있어요. 제 IP는 172.16.10.50예요." (응답)
```

카메라의 IP를 몰라도, 브로드캐스트는 **같은 네트워크 대역 전체에 뿌려지기 때문에** 카메라가 응답할 수 있다.

### 랜선을 직접 연결해야 하는 이유

브로드캐스트는 **같은 네트워크 안에서만 전달**된다.
라우터(공유기)는 브로드캐스트를 외부로 넘기지 않는다. 즉, 내부망과 격리된 카메라에는 브로드캐스트가 닿지 않는다.

랜선을 노트북에 직접 연결하면, 노트북과 카메라가 같은 네트워크에 속하게 되어
탐색 프로그램의 브로드캐스트가 카메라까지 도달한다.

```
내부망에 연결된 상태:
[노트북] → [공유기/스위치] → (브로드캐스트 차단) → 카메라에 도달 안 함

랜선 직접 연결:
[노트북] ←—— 랜선 ——→ [카메라]  ← 브로드캐스트 도달, IP 확인 가능
```

이것이 동료가 말한 **"PoE 상단 연결을 제거하고"** 의 의미이기도 하다.
상단 연결(내부망)을 끊어야 노트북과 카메라가 같은 단독 네트워크를 이루게 된다.

## 정리

| 개념 | 한 줄 요약 |
|------|-----------|
| IP 주소 | 장치의 고유 주소 |
| Subnet Mask | 같은 네트워크인지 판별하는 기준 |
| Gateway | 다른 네트워크로 나가는 출구 |
| IP 변경 이유 | 접근 가능한 대역으로 맞추거나, 외부 접속 경로를 열기 위해 |
| 브로드캐스트 탐색 | 같은 네트워크에 뿌려서 IP 모르는 장치도 찾아내는 방식 |

IP 설정은 단순히 숫자 바꾸는 게 아니라, **어떤 네트워크에 속할지 결정하는 일**이다.
카메라가 보이지 않는다면, 먼저 대역이 일치하는지부터 확인하는 게 순서다.
