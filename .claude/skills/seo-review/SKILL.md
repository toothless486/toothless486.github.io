# SEO 검증 스킬

기술 블로그 포스트 또는 사이트 설정을 입력받아 SEO 최적화 상태를 항목별로 직접 판정(✅ / ⚠️ / ❓)하고 개선 제안을 출력한다.

> 참고: 재그지그의 개발 블로그 — [기술 블로그를 위한 SEO](https://wormwlrm.github.io/2023/05/07/SEO-for-Technical-Blog.html)
> 일반적인 SEO 팁 나열이 아닌, 개인 기술 블로그(정적 사이트)에 실제 적용 가능한 기술 항목을 기준으로 판정한다.

---

## 판정 기호

| 기호 | 의미 |
|------|------|
| `✅` | 확인됨 — 충족 |
| `⚠️` | 미흡하거나 누락됨 — 개선 필요 |
| `❓` | 파일/설정만으로 판단 불가 — 자가 점검 필요 |

---

## 검증 범위

입력 유형에 따라 판정 범위가 달라진다.

- **포스트 파일 (`.md`)**: Front matter + 본문 콘텐츠 기준 판정
- **사이트 설정 (`_config.yml` 등)**: 전역 SEO 설정 기준 판정
- **HTML 소스**: 렌더링된 메타태그 기준 판정
- **복합 입력**: 가능한 모든 항목 판정

---

## 체크리스트 항목

### 1. Title 태그 — 검색 결과 제목
- 페이지마다 고유한 title이 설정되어 있는가
- title이 주요 키워드를 포함하는가
- title 길이가 적절한가 (30~60자, 영문 기준 60자 이하)
- 포스트 제목과 title이 일치하거나 유사한가
- 사이트명이 title에 포함되되 키워드 뒤에 위치하는가 (예: `포스트 제목 | 블로그명`)

### 2. 메타 디스크립션 — 검색 결과 설명
- 각 포스트에 고유한 메타 디스크립션이 있는가
- 디스크립션 길이가 적절한가 (80~160자)
- 디스크립션이 포스트 핵심 내용을 요약하는가
- 주요 키워드가 자연스럽게 포함되어 있는가
- front matter에 `description` 또는 `excerpt` 필드가 있는가

### 3. Canonical URL — 중복 콘텐츠 방지
- 각 페이지에 canonical 링크가 설정되어 있는가
- canonical URL이 실제 접근 가능한 URL과 일치하는가
- 페이지네이션·태그·카테고리 페이지에서 중복 canonical 문제가 없는가
- `https`로 통일되어 있는가 (http → https 리다이렉트 포함)

### 4. Open Graph — SNS 공유 최적화
- `og:title`이 설정되어 있는가
- `og:description`이 설정되어 있는가
- `og:image`가 설정되어 있는가 (권장 크기: 1200×630px)
- `og:url`이 절대 경로로 설정되어 있는가
- `og:type`이 포스트는 `article`, 홈은 `website`로 구분되어 있는가
- `article:published_time`, `article:author` 등 article 메타가 있는가

### 5. Twitter Card — 트위터/X 공유 최적화
- `twitter:card`가 설정되어 있는가 (`summary_large_image` 권장)
- `twitter:title`, `twitter:description`, `twitter:image`가 있는가
- `twitter:site` 또는 `twitter:creator`로 계정이 연결되어 있는가

### 6. Sitemap — 크롤링 효율
- `sitemap.xml`이 생성되어 있는가
- sitemap이 Google Search Console에 제출되어 있는가
- 불필요한 페이지(태그, 카테고리 중복 등)가 sitemap에서 제외되어 있는가
- `lastmod` 날짜가 포함되어 있는가

### 7. robots.txt — 크롤링 제어
- `robots.txt`가 존재하는가
- `robots.txt`에 sitemap 위치가 명시되어 있는가
- 크롤링 허용/차단 범위가 의도대로 설정되어 있는가
- 특정 봇(Googlebot 등)에 대한 설정이 적절한가

### 8. 구조화 데이터 (JSON-LD) — 리치 스니펫
- JSON-LD 형식의 구조화 데이터가 포함되어 있는가
- `BlogPosting` 또는 `Article` 스키마가 적용되어 있는가
- `headline`, `datePublished`, `author`, `image` 필드가 있는가
- Google의 리치 결과 테스트(rich results test)로 검증됐는가

### 9. 콘텐츠 — 검색 의도 충족
- 포스트 제목(H1)이 하나만 존재하는가
- 소제목이 H2 → H3 순서로 계층적으로 구성되어 있는가
- 핵심 키워드가 제목, 첫 단락, 본문에 자연스럽게 분포하는가
- 이미지에 `alt` 속성이 설정되어 있는가
- 내부 링크(관련 포스트 연결)가 포함되어 있는가
- 외부 링크에 적절한 `rel` 속성이 있는가 (`noopener`, `noreferrer` 등)

### 10. 성능 및 기술 환경
- 페이지 로딩 속도가 빠른가 (Core Web Vitals 기준)
- 이미지가 최적화되어 있는가 (WebP 형식, 적절한 크기)
- 모바일 반응형 디자인이 적용되어 있는가
- HTTPS가 적용되어 있는가
- `hreflang`이 다국어 콘텐츠에 설정되어 있는가 (해당 시)

### 11. Google Search Console 연동
- Search Console에 사이트가 등록되어 있는가
- 소유권 인증이 완료되어 있는가
- 색인 오류(Index Coverage) 이슈가 없는가
- 검색 성능(노출수, 클릭률, 순위)을 주기적으로 모니터링하고 있는가

## 출력 형식

아래 형식을 그대로 사용한다. `⚠️` 항목에만 괄호 안에 한 줄 이유를 추가하고, `✅`·`❓`에는 설명을 붙이지 않는다.

```markdown
## SEO 검증 결과

**검증 대상**: [포스트명 또는 파일명]
**한 줄 평**: [전체적인 SEO 상태를 한 문장으로]

---

### 1. Title 태그
- ✅ 페이지마다 고유한 title이 설정되어 있는가
- ⚠️ title 길이가 적절한가 (현재 72자 — 60자 이하 권장)
- ✅ 포스트 제목과 title이 일치하거나 유사한가
...

### 2. 메타 디스크립션
...

### 우선순위 개선 제안

1. **og:image 추가**: SNS 공유 시 미리보기 이미지가 없으면 클릭률이 크게 낮아짐. 포스트별 대표 이미지를 1200×630px로 설정 권장
2. **메타 디스크립션 작성**: front matter에 `description` 필드 추가 — 검색 결과 클릭률에 직접 영향
3. **JSON-LD 구조화 데이터 추가**: `BlogPosting` 스키마로 리치 스니펫 노출 가능성 확보
```
